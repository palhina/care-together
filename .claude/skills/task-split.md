# /task-split スキル

機能や要件をもとに、機能ごとに`/tickets`配下にチケットを分割します。

## 実行内容

1. 要件を分析し、独立した実装単位に分割
2. `/tickets`ディレクトリ配下にチケットファイルを作成
3. ファイル名は連番プレフィックス付き（例: `001-auth.md`, `002-onboarding.md`）
4. 各チケットファイルでTODOも管理

## チケットファイルのフォーマット

```markdown
# [チケット名]

## 概要
[このチケットで実装する内容の概要]

## 関連画面
- [関連する画面パス]

## 実装タスク
- [ ] タスク1
- [ ] タスク2
- [ ] タスク3

## 受け入れ条件
- [条件1]
- [条件2]

## 備考
[注意点や依存関係など]
```

## 使用例

ユーザー: 「認証機能を実装したい」

→ `/tickets/001-auth.md` を作成:
```markdown
# 認証機能

## 概要
Apple Sign In / Google Sign Inによる認証機能を実装

## 関連画面
- /auth/login

## 実装タスク
- [ ] Supabase Auth設定
- [ ] Apple Sign In実装
- [ ] Google Sign In実装
- [ ] 認証状態管理（useAuth hook）
- [ ] ログアウト機能

## 受け入れ条件
- Apple IDでログインできる
- Googleアカウントでログインできる
- 認証状態が永続化される

## 備考
- iOS App Store申請にはApple Sign Inが必須
```
